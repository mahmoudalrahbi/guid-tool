Design Description

A dense, dark, no-chrome utility surface — built to live inside Chrome's 360-pixel side panel without ever competing with the page the Recorder is actually documenting.

Visual Hierarchy

Each surface stacks three horizontal regions: a persistent header (what Guide is this?), a scrolling work area (the Steps), and a fixed footer with the only two actions that matter mid-session — Pause/Resume and Complete Capture. Anchoring Complete Capture to the bottom means it never moves as Steps stream in.

Inside the work area, Steps follow a strict left-rail pattern: a monospace step number, then a 16:10 Annotated Screenshot, then the auto-generated description. The most recent Step is promoted with an amber number and an amber-edged thumbnail — the Recorder always knows what was just captured without scanning.

The Amber Accent

#F59E0B is treated as a signal color, not decoration. It appears in exactly four roles:

  • The pulsing dot in the status row — the only thing in motion when Recording is active.
  • The highlight ring drawn on every Annotated Screenshot — the product's literal value proposition.
  • The primary CTA (Complete Capture, Resume Recording, Start Recording) — never two amber buttons in one surface.
  • The number and border of the latest Step, plus the Guide History chevron on hover.

Hold the line on this. The moment a secondary button or a meta label borrows amber, the recording indicator stops reading as urgent.

360-Pixel Layout

At 360px, padding is the enemy. The side panel uses 16px horizontal gutters, a single column for Steps, and never two side-by-side controls in the footer unless the primary is full-width. Step descriptions are 12.5px Inter — small enough to fit a sentence-plus-URL per Step, large enough that the Recorder can sanity-check the auto-generated copy without zooming.

The Annotated Screenshot is the densest pixel in the surface, so it earns the most width. Step number, hover actions, and metadata orbit around it.

Implied Interaction Patterns

  • Recording dot uses a CSS @keyframes pulse — a soft 1.6s amber ring breath. Stops cleanly when paused; the dot itself desaturates to neutral grey so the surface is unambiguous at a glance.
  • Step rows reveal inline Edit and Delete affordances on hover — the Recorder can fix a wrong description or drop a misclicked Step without leaving the side panel.
  • Pause ⇄ Resume swap the footer's primary slot. In Paused state, Resume becomes the amber CTA and Complete shrinks to secondary — the next obvious action is to resume, not to ship a half-finished Guide.
  • Guide History rows shift their chevron 2px right on hover. Clicking opens the Guide in the Editor tab.
  • Empty state uses an amber cursor glyph inside a dashed-border tile, then three numbered hints. The Complete Capture button is rendered but disabled — present, not absent, so the Recorder learns the shape of the workflow before producing their first Step.

Watch Out For (Implementation Notes)

  • Side panel width is not 360 everywhere. Users can drag it wider. Build the layout fluid — the Step thumbnail uses aspect-ratio: 16/10 and the description uses min-width: 0 so long URLs truncate cleanly at any width. Don't hard-code 360.
  • The Annotated Screenshot is a real PNG, not a placeholder. The thumbnails in this mock are stylized SVG stand-ins. In production, the highlight circle must be rendered by drawing onto the captured image (Canvas) at capture time — never as a CSS overlay on top of an <img>, because exported HTML/Markdown for the Reader won't carry arbitrary CSS reliably.
  • Auto-generated descriptions can be wrong or empty. The hover-edit affordance is not optional. Plan for a Step to land with description = "" (e.g. click on an element with no accessible name) — render a muted placeholder like "Describe this Step" so the row is still tappable, never collapsed.
  • Timer drift across pause/resume. The status row shows 02:14 in both Active and Paused states — store activeMs, not startedAt, and freeze the value on pause. Otherwise a long pause silently inflates the displayed duration.
  • IndexedDB quotas. The popup footer shows 2.4 MB · IndexedDB as a quiet reassurance, but at ~50–150 KB per Step, a single 20-step Guide is non-trivial. Surface a warning state (amber text in this slot) when total storage crosses ~80% of the quota — the Recorder will silently lose Guides otherwise.
  • Keyboard shortcut shown on the Start button (⌘⇧R) implies a Chrome commands binding in manifest.json. Chrome can refuse the binding if another extension claimed it; render the kbd hint conditionally based on the resolved shortcut, or hide it when unset.
  • Side panel persistence. A Recording Session must survive tab switches and the side panel being closed and reopened — drive all rendering from the background service worker's state, not from in-panel React state alone.
  • Complete Capture in Paused state. Kept accessible (as a secondary button) — Recorders sometimes pause specifically because they're done. Don't make them resume just to finish.
