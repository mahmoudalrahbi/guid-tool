Design handoff — Editor

What to build, in order

Everything below maps 1:1 to the surface above. Selectors, tokens, and behaviors are stable — lift them directly.

1 · Design tokens

Centralized as CSS variables on :root. Don't hardcode colors anywhere else.

--bg-2#1A1A1Acard / chip
--bg-3#242424hover
--line#2A2A2Aborders
--text-1#ECECECprimary
--text-2#9A9A9Asecondary
--text-3#5E5E5Etertiary / mono
--text-4#3F3F3Fplaceholder

Typography

  • Sans — Inter (400/500/600/700), system-ui fallback
  • Mono — JetBrains Mono (URLs, eyebrows, hex, metadata)
  • Title — 38px / 700 / -0.025em tracking
  • Body / step desc — 15px / 1.55 line-height
  • Meta / mono — 11–12px, uppercase 0.1em on eyebrows

Spacing & radii

  • Doc column max-width 880px, 32px horizontal padding
  • Step row vertical gap 8px, internal gap 12–14px
  • Radii: 7px (buttons/chips), 8–9px (inputs), 10–12px (cards), 14–16px (panels)
  • Hairline borders only: 1px solid var(--line) — no shadows except top bar & toasts

2 · Top bar (.topbar)

Sticky, 60px tall, translucent rgba(15,15,15,0.85) with backdrop-filter: blur(12px). Three-column grid: brand · save status · actions.

  • Title is editable in-place via #titleInput. input event triggers flashSaving(); debounce 700ms before flipping back to "Saved." Persist to IndexedDB on every flip-to-saved — not on every keystroke.
  • Save status = single .save-status node toggling .saving. Green pip for Saved, accent pulsing pip for Saving. The "· just now" timestamp should update with the same cadence ("just now" → "a minute ago" → "10:42").
  • Export dropdown opens on click, closes on outside click or Escape. Each .item carries data-fmt="html|pdf|docx|md|copy" — wire each to its exporter. Show a toast on success.

3 · Guide title & description

  • #titleInput is an <input>, not a contenteditable, so iOS keyboards behave. Bottom border underlines on hover (line) and on focus (accent).
  • #descInput is a <textarea> with autoSize() bound to input. Empty state shows the long placeholder — don't truncate it. Same autosave behavior as title.
  • Both fields contribute to the exported file's frontmatter (HTML <title> + meta description, PDF cover, DOCX heading + intro paragraph, Markdown # title + blockquote).

4 · Step row (.step)

Two-column grid: 28px rail + flexible body. The whole row is draggable="true".

  • Rail stacks the drag handle (opacity:0, fades in on row hover) and the step number badge (.step-num, 28×28, mono 12).
  • Screenshot is a fixed 16:9 wrapper. Render the click highlight as a baked-in part of the PNG — don't overlay it in CSS, exports won't carry it.
  • Description = autosizing textarea. Background transparent at rest, --bg-2 on hover, accent border on focus. Two-line minimum height.
  • URL footer shows the source page in mono 11px, truncated with ellipsis if too long. Optional — driven by a settings toggle.
  • Hover toolbar (top right): Duplicate, Re-annotate, Delete. Delete button gets red-tinted hover. opacity:0 at rest.

5 · Reorder, insert, delete

  • Reorder: HTML5 drag & drop. On dragstart add .dragging (40% opacity). On dragover add .drag-over to the target row — CSS draws a 2px accent line at the row's top via ::before. On drop, splice the dragged node (and its trailing .insert-gap) before/after target. Call renumber() + flashSaving().
  • Insert gap: zero-height spacer between rows; hovering it reveals a pill button to insert a manual step at that position.
  • Delete: remove the row + one adjacent gap. Show a .toast with an Undo button (3.8s lifetime). Undo restores the row at its original index. Renumber after every change.
  • Renumber: renumber() reads all .step children of #stepsList, sets each .step-num to String(i+1).padStart(2,'0'), updates #stepCount, and flips data-empty on body when count hits zero.
  • Empty trigger: when the last step is deleted, body[data-empty="true"] hides .has-steps and reveals .empty-state via CSS — no JS swap needed.

6 · Empty state (.empty-state)

  • Centered card, 560px max-width, dashed border, dotted-fill cursor glyph with the accent fill.
  • Two CTAs only: Start new Recording (primary, opens a new Recording Session) and Open Guide History (secondary, opens the popup history view).
  • Keep the title and description fields visible — the user may still want to rename a now-empty Guide. The Empty state replaces the step list, not the whole document.

7 · Autosave contract

  • Single source of truth: guides object store in IndexedDB, keyed by guideId.
  • Debounce 700ms on title / description / step description edits. Persist immediately on structural changes: reorder, delete, undo, insert, manual add.
  • Save status UI lags the actual persistence by one tick — flip to "Saving…" on input, flip to "Saved" only after the IDB transaction's oncomplete. If onerror, surface a red "Couldn't save" state in the same slot.
  • Editor is local-only — no server, no fetch, no telemetry. Treat the file as the entire universe of state.

8 · Watch out for

  • Textarea height jumps on input — always call autoSize() right after setting value programmatically (e.g. after undo restore), or rows stay at the original height.
  • Drag handle on touch — HTML5 drag-and-drop doesn't fire on mobile Safari. If touch support matters, swap in a pointer-events-based implementation; otherwise hide the handle below 700px and rely on a long-press menu with "Move up / Move down."
  • Export with very large guides — PDF and DOCX with 50+ embedded screenshots can OOM the renderer. Stream pages or warn the Recorder at >30 steps.
  • Undo window is 3.8s and not stacked. If the user deletes a second step before the first toast expires, the first toast's Undo is gone. Decide whether to stack or drop earlier toasts; current design drops.
  • Accent is a signal color. Only four roles: save-status pulse, click highlight in the screenshot, primary CTAs, focus rings on editable text fields. Don't add a fifth.
