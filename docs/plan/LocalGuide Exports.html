Export handoff

All three exports are generated locally in the Editor tab — no server. Each format is a transform over the same in-memory Guide object: { title, description, recordedAt, source, steps: [{ id, description, screenshotPng, url }] }.

PDF · .pdf

Fixed layout, printable

For the Reader who needs a sharable artifact — onboarding handout, audit trail, attached-to-ticket.

Library

  • pdf-lib (recommended) — works in the browser, MIT, ~100KB. Build pages programmatically from the Guide object.
  • Alternative: window.print() against a styled HTML doc, then user saves as PDF. Free but loses TOC page numbers.

Structure

  • Page 1 = cover: title (Source Serif 4 36pt), description, meta block, TOC with dot leaders + page numbers.
  • Pages 2..N = steps: STEP NN eyebrow, step title (serif 15pt), screenshot (16:9, 1px gray border), URL footer.
  • Running header on every page: amber dot + "LocalGuide" + page number.
  • Avoid splitting a step across pages — use page-break-inside: avoid equivalent.

Watch out

  • Embed screenshots at 2× resolution so they don't pixelate on Retina print.
  • Don't render with system fonts — embed Inter and Source Serif in the PDF so it looks identical on any reader.
DOCX · .docx

Editable in Word

For the Reader who needs to localize the steps, add company branding, or extend the guide inside an existing template.

Library

  • docx (npm) — pure JS, builds OOXML in browser. Use Document, Paragraph, HeadingLevel.HEADING_1/2, ImageRun.

Structure

  • Title → Heading 1 style. Description → italic body paragraph below it.
  • Metadata → 1-column borderless table (Recorded / Source / Steps).
  • Per step → Heading 2 "STEP NN — title", body paragraph, inline image, caption paragraph styled as Caption.
  • Use real Word styles (Heading 1, Heading 2, Caption) — not just bold text. That's what lets the Reader restyle the whole doc by editing the template.

Watch out

  • Images: convert PNG to JPEG ≥ 1280px wide before embedding; DOCX inflates fast.
  • The amber badge can be a "Heading 2 numbered list" or a shaded run — both render in Word, but the shaded run survives "Clear formatting" cleaner.
Markdown · .md

Source-controllable

For the Reader who lives in GitHub, GitLab, Notion, or Obsidian. Diffs cleanly, embeds anywhere, future-proof.

Library

  • None — just string concatenation. The whole exporter is < 60 lines.

Structure

  • Top of file: HTML comment with localguide version + recorded date.
  • # Title, then a > blockquote for the description.
  • Per step: ## Step N — title, body paragraph, image, then Source: app.com/path line. Separate with ---.
  • Images go in a sibling ./assets/step-NN.png path inside the exported .zip. Reference them with relative paths so the file works offline.

Watch out

  • Export as a zip when there are images. A bare .md with broken ./assets/ references is the most common Reader complaint.
  • Escape pipes, asterisks, and underscores in auto-generated descriptions or GitHub's renderer mangles them.

Shared across all three formats

Filename
slugify(guide.title) + '-' + dateISO + '.' + ext — e.g. how-to-invite-a-teammate-2026-05-23.pdf. Truncate slug to 60 chars.
Click highlight
Already baked into the screenshot PNG by the Recording Session (amber circle on a transparent overlay layer, flattened at capture). Exporters should never re-draw it — they have no idea where the click happened.
Accent color
#F59E0B in all three formats — match the Editor. Use it for the title underline (DOCX), TOC accent line (PDF), and STEP NN badge (DOCX + MD preview). Nothing else.
Typography
PDF and DOCX use a serif (Source Serif 4 or Georgia fallback) for titles and step headings; sans (Inter / Calibri fallback) for body. Markdown inherits whatever the renderer applies — don't fight it.
Privacy footer
PDF cover only: small amber "● Recorded locally · Never uploaded" mark. Reinforces the product's promise on the artifact itself, which is the only piece the Reader sees.
Empty guide
If the Recorder hits Export on a Guide with zero steps, surface a confirm in the Editor — don't ship an empty file.
